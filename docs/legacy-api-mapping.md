# Legacy API Mapping (privchat-sdk -> privchat-rust)

本文档是执行级映射，不是设计草案。目标是把旧 `privchat-sdk` 的能力迁移到新 `privchat-rust`，并标注迁移中做的优化。

## 1. Session / Auth / Bootstrap

| Legacy API | New API | Status | Optimization |
|---|---|---|---|
| `connect` | `PrivchatClient.connect` | DONE | 统一 UniFFI async 导出，移除同步桥接 |
| `disconnect` | `PrivchatClient.disconnect` | DONE | Actor 串行执行，避免锁抢占 |
| `login` | `PrivchatClient.login` | DONE | 状态机约束顺序（Connected -> LoggedIn） |
| `register` | `PrivchatClient.register` | DONE | 同上 |
| `authenticate` | `PrivchatClient.authenticate` | DONE | 状态机约束顺序（LoggedIn -> Authenticated） |
| `run_bootstrap_sync` | `PrivchatClient.run_bootstrap_sync` | DONE | local-first 强门禁 |
| `is_bootstrap_completed` | `PrivchatClient.is_bootstrap_completed` | DONE | 单一事实源来自 actor state |
| `shutdown` | `PrivchatClient.shutdown` | DONE | TaskRegistry 统一回收 |

## 2. Local-first Storage

| Legacy API Class | New API Class | Status | Optimization |
|---|---|---|---|
| message CRUD | `create_local_message/get/list/update...` | DONE | 本地唯一主键固定 `message.id` |
| channel CRUD | `upsert_channel/get/list/...` | DONE | actor 串行 DB 调度 |
| user/friend/group local tables | `upsert/get/list...` | DONE | per-user 目录隔离 |
| reaction/mention/reminder | 对应 local APIs | DONE | 明确读写路径与幂等更新 |
| kv | `kv_put/kv_get` | DONE | 与主库职责分离 |

## 3. Queue and ID Semantics

| Requirement | New Behavior | Status | Optimization |
|---|---|---|---|
| normal queue | `enqueue/peek/ack_outbound_messages` | DONE | actor 内串行避免并发错序 |
| file multi-queue | `enqueue/peek/ack_outbound_files` | DONE | route-key 选择多队列 |
| `local_message_id` generated by snowflake | enqueue 时写回 DB | DONE | 保留服务端 dedupe 语义 |
| local CRUD uses `message.id` only | 全部 local API 仅收/返 `message.id` | DONE | 降低本地对象歧义 |

## 4. RPC Bridge and Remote Wrappers

## 4.1 Generic bridge

- `rpc_call(route, body_json)` -> DONE
- `entity_sync_remote(payload_json)` -> DONE

Optimization:
- 所有远端封装统一走同一路径：`PrivchatSdk::rpc_call`（Actor 内执行）
- 不引入额外 runtime/线程模型
